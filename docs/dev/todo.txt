Consider defining a Pydantic Model for AssetChanges:

Your return type:
dict[str, list[Asset]]

If this grows, you might consider defining a Pydantic model:

class AssetChanges(BaseModel):
    check_ins: list[Asset]
    check_outs: list[Asset]

Then your function becomes:

def detect_asset_changes(
    previous: dict[int, Asset],
    current: dict[int, Asset],
) -> AssetChanges:

and returns:

return AssetChanges(
    check_ins=[previous[id] for id in check_in_ids],
    check_outs=[current[id] for id in check_out_ids],
)

This is more self-documenting and gives you autocompletion
(changes.check_ins instead of changes["check_ins"]). For a small project,
your dictionary approach is perfectly reasonable, but a dedicated model
becomes increasingly valuable as the project grows.

request_events_report() -> int
